# Навык UI/UX и стилизации

При разработке пользовательских интерфейсов в этом проекте, ВСЕГДА следуй этим правилам:

1. **Технологический стек**:
   - Используй **Vanilla CSS** (или CSS Modules, если используется фреймворк вроде Next.js/Vite) для максимального контроля и гибкости.
   - Избегай использования сторонних CSS-фреймворков (например, TailwindCSS), если только пользователь явно не попросил об этом в конкретной задаче.

2. **Современная эстетика и "Wow-эффект"**:
   - Интерфейс должен выглядеть премиально и современно с первого взгляда. Избегай простых минималистичных решений (MVP style), если не указано иное.
   - **Цветовая палитра**: Не используй базовые цвета браузера (просто red, blue, green). Используй гармоничные, специально подобранные палитры (например, HSL), градиенты, эффекты полупрозрачности (glassmorphism) и проработанные светлые/темные темы.
   - **Типографика**: Используй современные шрифты (например, Inter, Roboto, Outfit из Google Fonts). Не используй шрифты по умолчанию.

3. **Динамичность и интерактивность**:
   - Интерфейс должен ощущаться "живым". Обязательно добавляй эффекты при наведении (hover states), фокусе и нажатии на интерактивные элементы.
   - Используй микро-анимации (плавные переходы, `transition`) для улучшения пользовательского опыта (UX).

4. **Доступность и семантика (HTML/SEO)**:
   - Используй правильные семантические теги HTML5 (`<header>`, `<main>`, `<article>`, `<nav>`, `<section>`).
   - У каждого интерактивного элемента должен быть уникальный и понятный атрибут `id` (это также полезно для тестирования).
   - Следи за наличием правильной иерархии заголовков (один `<h1>` на страницу) и атрибутов `alt` для изображений.

5. **Демонстрация вместо заглушек**:
   - Если для верстки интерфейса нужны изображения, не используй серые блоки-заглушки. Генерируй или подбирай демо-изображения для создания полноценного визуального представления.

6. **Структура компонентов**:
   - Выноси повторяющиеся стили в константы или общие CSS-классы. Сохраняй компоненты переиспользуемыми и независимыми друг от друга.

7. **Язык общения**:
   - При обсуждении дизайна и предложении макетов общайся на русском языке.
