# Навык работы с Git

При выполнении любых операций с Git в этом проекте, ВСЕГДА следуй этим правилам:

1. **Сбор информации**: Перед любым коммитом всегда выполняй команду `git status`, чтобы увидеть, какие файлы подготовлены, и `git diff`, чтобы проверить фактические изменения в тексте.
2. **Атомарные коммиты**: Группируй связанные изменения в один коммит. Не смешивай независимые изменения (например, исправление ошибки и добавление новой не связанной функции) в один коммит.
3. **Сообщения коммитов**: Используй [Conventional Commits](https://www.conventionalcommits.org/).
   - Формат: `<тип>(<область>): <описание на русском или английском>`
   - Типы:
     - `feat`: Новая функция
     - `fix`: Исправление ошибки
     - `docs`: Изменения только в документации
     - `style`: Форматирование, пропущенные точки с запятой и т.д.
     - `refactor`: Рефакторинг рабочего кода
     - `test`: Добавление недостающих тестов, рефакторинг тестов
     - `chore`: Обновление задач сборки, конфигов пакетного менеджера и т.д.
   - Описание должно быть понятным и отражать суть изменений. Мы используем русский язык для общения, поэтому описания могут быть на русском языке.
4. **Безопасность**: Никогда не используй force push (`--force` или `-f`) без явного разрешения пользователя.
