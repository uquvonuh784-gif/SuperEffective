# Product Requirements Document (PRD): Next-Gen Task Tracker

## 1. Описание Проекта (Product Vision)
Гибридное рабочее пространство, объединяющее функциональность создания заметок (Notes) и управления задачами (Tasks) с геймификацией и AI-ассистентом. Платформа предназначена для команд и семей, позволяя превращать идеи в структурированные задачи, отслеживать прогресс и мотивировать пользователей через систему баллов. Основной упор делается на надежность и умное управление временем с помощью проактивного AI.

## 2. Основные Сущности (Core Entities)
### 2.1 Унифицированная Нода (Node: Заметка / Задача)
Все начинается с заметки (Markdown-документ). У любой ноды есть системные даты создания (`created_at`) и обновления (`updated_at`). Любая заметка может быть преобразована в Задачу путем добавления следующих свойств (Properties) через UI:
- **Свойства (Metadata):** Статус (`To Do`, `In Progress`, `Done`, `Archived`), Дедлайн (Date — по умолчанию сегодняшний день), Исполнитель (Assignee), Приоритет (Low, Medium, High).
- **Связи (Backlinks):** Двунаправленные ссылки на другие ноды (проекты, идеи).
- **Награда (Reward Points - RP):** Количество баллов (вес задачи), начисляемых за выполнение.

### 2.2 Пользователи и Пространства (Users & Workspaces)
- **Workspaces:** Разделение на Личные (Personal) и Общие (Shared - семья, отдел).
- **Роли:** Владелец, Редактор, Читатель.

## 3. Ключевые Фичи (MVP & Beyond)
### 3.1 Текстовый Редактор (Obsidian/Notion Style)
- Поддержка Markdown.
- Блочный редактор для создания задач, таблиц, чек-листов.
- Встраивание свойств задачи в начало документа (Frontmatter/Properties view).

### 3.2 Геймификация
- Назначение баллов (RP - Reward Points) при создании задачи по уровню сложности.
- Дашборд прогресса: Личный и Командный топ по заработанным баллам (за день, неделю, месяц).
- Система "Ачивок" и множителей за "Серию" (выполнение задач несколько дней подряд).
- **Штрафы:** Списание баллов, если задача возвращается из статуса `Done` в работу.

### 3.3 Интеграция с Google Drive
- Синхронизация документов и вложений с Google Drive.
- Возможность прикреплять файлы из Google Drive напрямую к задачам и заметкам.
- Резервное копирование (бэкап) структуры графа и базы знаний пользователя для надежности.

### 3.4 AI-Копилот (AI Agent)
- Разбор "сырых" заметок/брейнштормов и их декомпозиция на конкретные задачи.
- Фоновая AI-генерация свойств для новых задач на основе контекста (автовыбор дедлайна, приоритета).
- **Проактивные напоминания:** Периодический анализ задач (Cron-задачи). AI формирует сводки по задачам, по которым давно нет активности (по `updated_at`), или чьи сроки поджимают.

### 3.5 Граф связей и Архивация
- Визуализация взаимосвязей между идеями, конспектами и задачами для лучшего понимания структуры проекта.
- Задачи со статусом `Archived` скрываются из Канбана и Дашбордов, но остаются доступными на Графе и в поиске.

## 4. Технический Стэк
- **Фронтенд:** Next.js (App Router), React, Vanilla CSS для точечного контроля и премиального glassmorphism UI (согласно навыку `ui_ux`).
- **Стэйт-менеджмент & Data Fetching:** React Query / SWR / Supabase Realtime (для совместной работы).
- **Текстовый Редактор:** Tiptap / BlockNote.
- **Бэкенд и БД:** Supabase (PostgreSQL) — для аутентификации и реляционного хранения графов и задач (Унифицированная нода).
- **AI Интеграция:** Groq API (Llama 3 Modelle).

## 5. Ближайшие Шаги по Реализации (Roadmap)
1. **Этап 1: Структура БД и Типизация** (Настройка Supabase, создание схем сущностей Заметок и Задач с учетом `created_at`, `updated_at` и статуса `Archived`).
2. **Этап 2: Базовый Редактор и UI** (Интеграция Tiptap, создание сайдбара и компонентов в стиле premium glassmorphism).
3. **Этап 3: Система свойств и Канбан** (Добавление метаданных к заметкам, система статусов, скрытие архива).
4. **Этап 4: Google Drive и Геймификация** (Интеграция с Диском для бэкапов, логика начисления и списания баллов).
5. **Этап 5: Проактивный AI** (Подключение Groq API для декомпозиции и Cron-задач для напоминаний).
